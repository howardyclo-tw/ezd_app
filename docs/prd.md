## **專案定位**

- 熱舞社的管理系統
- 目標
    - 短期（MVP: 讓人只做判斷，不要再做計算與整理)
        - 實質減少幹部行政時間
        - 消除高風險人工錯誤（名額、排序、計費）
        - 不改變課務與財務既有工作習慣
    - 中期（完整版本）
        - 集中化課程內容與公告
        - 降低跨角色溝通成本
        - 建立可擴充、可維護的系統邊界
- 使用者身份：幹部(也是社員), 社員, 非社員
- 社員有效期限管理（2026-03-28 新增）：
    - 採「年度群組」制度（如「2026」群組），每個群組有統一到期日
    - 成員歸屬一個群組，幹部修改群組到期日自動影響所有成員
    - 非社員不屬於任何群組
    - 匯入成員、購卡含社員費時自動加入最新群組

### 現況

1. 總人數/每期上課人數約100上下

2. 只使用 surveycake 搜集問卷,公司內部網路架設的wiki作為content management, email使用公司內部outlook

3. 副社長,財務,宣傳,班長之職務（跟處理課程金流相關的）副社長跟班長職位最為優先

4. 預算越低越好，一個人力而已，希望能快速打造MVP

**每年初/不定時招生**

1. 副社維護社員名單
     1. 現階段只有年初一次性招生(公司補助300元,社費只要繳1500）
     2. 之後每次報名課程階段，使用者填寫問卷時都可以順便報名「加入社團」，須額外付1800元
     3. ~~從每次報名階段結束後，副社需要更新社員名單（如果有新社員報名的話）~~ → **系統化後：購卡含社員費時自動加入最新年度群組，匯入成員時自動分配群組。**

**課程規劃階段**

1. 課務線下與老師敲定上課時程
2. 編寫年度行事曆
3. 課務在公司內部 wiki 上撰寫並公告課程內容（幹部審核後才會發布,有權限）
    1. 上課日期/時間, 時長, 堂數, 課名, 老師, 教室, 人數上限, 課程備註
    2. 可能會有某一堂因故停課的時候, 需要有彈性可以更改該堂上課日期
    3. 課程類型分為：(常態)試跳, 一般常態, ~~特殊常態~~, 風格體驗, (Workshop)專攻班, ~~團練~~, ~~表演班~~ （~~希望能抽象化, 變成可彈性設定的課程規則模板~~ → **系統實作為固定選項：normal, trial, style, workshop。特殊常態保留但隱藏，團練/表演班已移除。**）
        1. ~~(常態)試跳報名會排志願序, 會有優先度的排序~~ → **目前系統未實作志願序，試跳以先到先得方式報名。**
        2. ~~特殊常態：實驗性試開的常態課~~ → **已從 UI 隱藏，DB 保留但不可選。**
        3. 通常常態是固定每一期(2個月)，風格/專攻也會跟著常態一起規劃，但是風格/專攻有可能不定期舉辦(通常在每一期結束到下一期開始之間,或是課程期間空擋)

**堂卡制度（2026 新制，取代過去單堂報名制）**

1. 堂卡購買規則
    1. 取消過去單堂報名制度，全面改成「堂卡制」
    2. 購買最小單位：5 堂卡（可 config）
    3. 堂卡定價（可 config）：
        - 社員：單堂 270 元，5 堂卡 1,350 元
        - 非社員：單堂 370 元，5 堂卡 1,850 元
    4. 堂卡有效期限制度（2026-03-28 新增）：
        - 每批堂卡有獨立到期日（預設為購買當年度年底，幹部可自訂）
        - 堂卡必須在上課當天仍然有效才能使用（到期日 ≥ 上課日期）
        - 報名時系統自動優先使用最快到期的堂卡（FIFO）
        - 單堂報名：逐堂檢查，每堂用該堂日期匹配
        - 整期報名：以最後一堂日期檢查，堂卡須覆蓋整期
        - 幹部可在成員管理中新增堂卡並指定到期日
    5. 適用所有課程類型
2. 購買時段限制
    1. 採限時購買制：每兩個月開放一週購買（開放週期可 config）
    2. 非購買時段不可加購堂卡
    3. 幹部可手動開關購買時段（特殊情況）
3. 課程扣卡規則（每堂扣卡數可 config）
    1. 課程單堂收費以堂卡為單位
    2. 預設扣卡數：

            2. 一般常態課程：缺2堂，可以補特別常態課程1堂
            3. 竹北跟總部課程補課制度獨立：在竹北缺課無法到總部補課，反之亦然
        8. **補課額度計算規則（2026-03-28 系統實作）：**
            1. 可用補課 = min(未使用缺席數, 1/4 額度上限剩餘) + 幹部贈予額度
            2. 1/4 規則是「額度上限」，不是點數。必須有實際缺席才能使用
            3. 幹部贈予額度不需要缺席來源，可直接補課任何堂次
            4. 補課學員/轉入學員對補課堂次請假，不歸還補課額度
            5. 補課學員/轉入學員可對該堂次申請請假（釋出名額）
            6. 不能對已過去的堂次申請補課或請假
        9. **駁回歸還規則（2026-03-29 系統實作）：**
            1. 補課駁回（幹部贈予）：清除目標堂次出席紀錄，歸還 1 點 makeup_quota 至學員帳戶
            2. 補課駁回（缺席來源）：清除目標堂次出席紀錄，不歸還 makeup_quota（缺席仍在，學員可重新申請）
            2. 補課駁回後重新核准：恢復出席紀錄；幹部贈予的補課會再扣 1 點 makeup_quota
            3. 請假駁回：恢復學員原本身份（補課→makeup、轉入→transfer_in、一般→清除），無額度消耗
            4. 轉讓駁回：清除 transfer_out/transfer_in 出席紀錄，恢復原狀，無額度消耗
            5. 堂卡訂單駁回：訂單標記 rejected，未曾核發堂卡，重算餘額即可
